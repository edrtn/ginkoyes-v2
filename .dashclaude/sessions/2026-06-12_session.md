## Chronologie

### Correction sync pipeline (hérité de la session précédente)

1. **INSERT IGNORE INTO** — `sync.ts` avait le fix dans le source mais pas dans le JS compilé. Recompilé avec `npx tsc -p sync/tsconfig.service.json`.

2. **Bug readline** — `rl.resume()` appelé après fermeture de readline sur petits fichiers. Fix : tracking `flushing`/`closed`, defer `finalize()`.

3. **OOM ARTARTICLE** — 82K lignes, readline + mysql2 = fuite mémoire à 4GB. Fix : `readFileSync` + split pour fichiers < 400MB.

4. **OOM CSHTICKETL** — 917MB, readline inutilisable. Fix : lecteur chunk manuel :
   ```typescript
   const CHUNK = 1024 * 1024;
   const chunkBuf = Buffer.alloc(CHUNK);
   const fdRead = fs.openSync(tempOutFile, "r");
   let leftover = "";
   let fileOffset = 0;
   while ((bytesRead = fs.readSync(fdRead, chunkBuf, 0, CHUNK, fileOffset)) > 0) {
     const chunk = leftover + chunkBuf.toString("utf-8", 0, bytesRead);
     const lines = chunk.split(/\r?\n/);
     leftover = lines.pop() || "";
     for (const line of lines) { parseLine(line); ... }
     fileOffset += bytesRead;
   }
   ```

5. **Filtre 4 ans** — Tables avec WHERE clause :
   - CSHTICKET: `TKE_DATE >= '2022-01-01'`
   - CSHTICKETL: `TKL_TKEID IN (SELECT TKE_ID FROM CSHTICKET WHERE TKE_DATE >= '2022-01-01')`
   - NEGBL/NEGBLL, COMBCDE/COMBCDEL, RECBR/RECBRL : même pattern

6. **Timezone** — `new Date().toISOString()` → `new Date().toLocaleString("fr-FR", { timeZone: "Europe/Paris", hour12: false })`

**Cause** : entrée registre NSIS stale `fbdd69e1-e400-58d9-8d87-49a836a4add3` référençant `C:\Ginkoyes\gui\Uninstall Ginkoyes Serveur.exe` qui n'existait plus. L'installeur NSIS détecte cette entrée et tente de fermer l'app avant de réinstaller.
