# Rapport de session — 2026-06-09

## Tâches complétées

### 1. Création de l'application Ginkoyes V2
- **Next.js 16** (TypeScript, Tailwind CSS v4, App Router, Turbopack)
- **node-firebird** connexion directe à Firebird Docker (localhost:3050)
- **shadcn/ui** + **Recharts** pour l'UI

### 2. API Routes (8 endpoints)
- `GET /api/articles` — recherche avec filtres (nom, ref, chrono, code-barres, marque, rayon, famille, collection)
- `GET /api/articles/[id]` — fiche article complète (infos + collections + codes-barres + stock)
- `GET /api/articles/[id]/ventes` — historique des ventes
- `GET /api/articles/[id]/receptions` — historique des réceptions
- `GET /api/stock` — stock global avec filtres
- `GET /api/dashboard` — stats agrégées avec filtres date (CA, tickets, panier moyen, top articles, CA par rayon/marque, CA mensuel)
- `GET /api/filters` — listes de marques, rayons, familles pour les dropdowns
- `GET /api/stats` — statistiques annuelles (CA, panier moyen, indice de vente)

### 3. Pages
- **Dashboard** (`/`) — KPIs gradient, sélecteur de période, courbe CA mensuel, top 10, pie chart rayons, bar chart marques
- **Recherche articles** (`/articles`) — barre de recherche + filtres dynamiques, table résultats
- **Fiche article** (`/articles/[id]`) — header gradient, KPIs rapides, stock visuel par couleur/taille, onglets ventes/réceptions
- **Stock** (`/stock`) — vue globale avec filtres et totaux
- **Statistiques annuelles** (`/stats`) — CA, panier moyen, indice de vente par année avec graphiques et tableau détaillé (évolution %)

### 4. Optimisation performance
- Dashboard : 15s → 0.4s grâce aux filtres date SQL
- Requêtes filtrées par `TKE_DATE` sur toutes les stats dashboard
