### Fix scan reseau dashboard (Electron)
- **Probleme** : le pare-feu macOS bloque les connexions TCP sortantes (`EHOSTUNREACH`) depuis le binaire Electron non signe vers les machines distantes du reseau local
- **Cause racine** : `net.Socket` et `mysql.createConnection` dans le processus principal Electron sont bloques par le firewall macOS pour les hotes distants (localhost/self fonctionne)
- **Fix scan de port** : remplacement de `net.Socket` par `/usr/bin/nc -z` via `child_process.exec` â€” nc est un binaire systeme non bloque
- **Fix test MariaDB** : nouvelle fonction `testMariaDbViaChild()` qui spawne un processus `node` systeme pour faire la connexion (le binaire `node` n'est pas bloque)
- **Ameliorations** : support multi-subnets (`getLocalSubnets()`), logs de diagnostic `[scan]`, messages UI enrichis avec nombre de ports detectes

## Problemes rencontres

1. **`net.Socket` EHOSTUNREACH dans Electron** â€” macOS firewall bloque les connexions TCP du binaire Electron non signe â†’ contourne avec `nc -z` (child process)
2. **`arp -a` echoue dans Electron** â€” commande introuvable (PATH restreint) et `execSync` timeout â†’ abandonne au profit de `nc` pour chaque IP
3. **`node -e` avec evalTypeScript** â€” Node.js v26 traite `-e` comme TypeScript par defaut â†’ utilise `--input-type=commonjs`
4. **`process.execPath` = Electron** â€” dans Electron, `process.execPath` pointe vers le binaire Electron, pas node â†’ recherche du `node` systeme via PATH
5. **MariaDB local refuse `mac.home`** â€” le MariaDB du Mac refuse les connexions depuis son propre hostname â†’ normal, pas de grant pour ce host
