## Notes techniques importantes

### Ventes dans Ginkoia : 2 sources
- **`CSHTICKETL`** (via `CSHTICKET`) = ventes **caisse magasin**
- **`NEGBLL`** (via `NEGBL`) = ventes **internet / BL**
- **Toute requête de ventes doit faire un UNION ALL des deux sources** pour être complète

### Commandes vs Réceptions
- **`COMBCDE` / `COMBCDEL`** = commandes fournisseurs (ce qui a été commandé)
- **`RECBR` / `RECBRL`** = bons de réception (ce qui a été effectivement reçu)
- Pour l'analyse achat, la base doit être les **commandes**, pas les réceptions
- Les réceptions servent uniquement à la colonne "Reçu" (suivi livraison)

### Colonnes utiles COMBCDEL
`CDL_ID, CDL_CDEID, CDL_ARTID, CDL_TGFID, CDL_COUID, CDL_QTE, CDL_PXACHAT, CDL_PXVENTE, CDL_REMISE1/2/3, CDL_COLID`

### Colonnes utiles NEGBLL (BL internet)
`BLL_ID, BLL_BLEID, BLL_ARTID, BLL_TGFID, BLL_COUID, BLL_QTE, BLL_PXBRUT, BLL_PXNET, BLL_PXNN`

### Correspondance colonnes Caisse ↔ BL
