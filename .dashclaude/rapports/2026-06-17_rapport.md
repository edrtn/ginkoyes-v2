### Fix scan reseau dashboard (Electron)
- **Probleme** : le pare-feu macOS bloque les connexions TCP sortantes (`EHOSTUNREACH`) depuis le binaire Electron non signe vers les machines distantes du reseau local
- **Cause racine** : `net.Socket` et `mysql.createConnection` dans le processus principal Electron sont bloques par le firewall macOS pour les hotes distants (localhost/self fonctionne)
- **Fix scan de port** : remplacement de `net.Socket` par `/usr/bin/nc -z` via `child_process.exec` — nc est un binaire systeme non bloque
- **Fix test MariaDB** : nouvelle fonction `testMariaDbViaChild()` qui spawne un processus `node` systeme pour faire la connexion (le binaire `node` n'est pas bloque)
- **Ameliorations** : support multi-subnets (`getLocalSubnets()`), logs de diagnostic `[scan]`, messages UI enrichis avec nombre de ports detectes
- **Resultat** : scan trouve 3 hotes (192.168.1.13, .67, .100), 2 serveurs MariaDB valides (82 651 articles)

## Problemes rencontres

1. **`net.Socket` EHOSTUNREACH dans Electron** — macOS firewall bloque les connexions TCP du binaire Electron non signe → contourne avec `nc -z` (child process)
2. **`arp -a` echoue dans Electron** — commande introuvable (PATH restreint) et `execSync` timeout → abandonne au profit de `nc` pour chaque IP
3. **`node -e` avec evalTypeScript** — Node.js v26 traite `-e` comme TypeScript par defaut → utilise `--input-type=commonjs`
4. **`process.execPath` = Electron** — dans Electron, `process.execPath` pointe vers le binaire Electron, pas node → recherche du `node` systeme via PATH
5. **MariaDB local refuse `mac.home`** — le MariaDB du Mac refuse les connexions depuis son propre hostname → normal, pas de grant pour ce host
