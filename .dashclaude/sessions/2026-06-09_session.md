# Session — 2026-06-09

## Contexte
Création de l'application web Ginkoyes V2 from scratch. Base Firebird Docker (localhost:3050, ginkoia.fdb).

## Commandes exécutées
- `npx create-next-app@latest` (en /tmp puis copie des fichiers)
- `npm install node-firebird recharts`
- `npx shadcn@latest init -d`
- `npx shadcn@latest add table card input badge separator tabs`
- `npx tsc --noEmit` → corrections TypeScript (recharts types)
- `npm run build` → succès

## Fichiers créés/modifiés

### Couche données
- `lib/db.ts` — Pool Firebird (5 connexions), helpers `query()` et `queryFirst()`
- `lib/queries.ts` — Toutes les requêtes SQL : recherche articles, fiche article, stock, ventes, réceptions, dashboard, filtres. Fonctions `buildArticleSearchWhere()` et `buildStockWhere()` pour les clauses WHERE dynamiques.

### API Routes
- `app/api/articles/route.ts` — Recherche articles avec filtres dynamiques
- `app/api/articles/[id]/route.ts` — Fiche article (parallel fetch: detail + collections + barcodes + stock)
- `app/api/articles/[id]/ventes/route.ts`
- `app/api/articles/[id]/receptions/route.ts`
- `app/api/stock/route.ts` — Stock global, GROUP BY article, filtrable
- `app/api/dashboard/route.ts` — Stats agrégées (parallel fetch: CA total, top articles, CA par rayon, CA par marque)
- `app/api/filters/route.ts` — Listes pour dropdowns

### Pages
- `app/layout.tsx` — Layout avec Navigation, title "Ginkoyes", lang="fr"
- `app/page.tsx` — Dashboard : 3 KPIs, top 10 articles, graphiques CA rayon (pie) + CA marque (bar)
- `app/articles/page.tsx` — Recherche avec SearchBar + filtres, table résultats
- `app/articles/[id]/page.tsx` — Fiche détaillée : cards infos, tabs stock/ventes/réceptions
- `app/stock/page.tsx` — Vue stock globale avec totaux

### Composants
- `components/Navigation.tsx` — Nav responsive avec highlight actif
- `components/SearchBar.tsx` — Input + dropdowns marque/rayon/famille, debounce 400ms
- `components/StockTable.tsx` — Table stock couleur/taille/qté/PUMP/valorisation avec totaux
- `components/SalesChart.tsx` — SimpleBarChart (horizontal) + SimplePieChart avec couleurs

## Décisions techniques
- `node-firebird` choisi car pure JS, pas de dépendances natives — compatible macOS sans compilation
- Connection pool de 5 connexions pour éviter les problèmes de concurrence
- Requêtes SQL avec `FIRST 50/100` pour limiter les résultats (pas de pagination serveur pour V1)
- Recherche avec debounce 400ms côté client
- Chargement lazy des onglets ventes/réceptions (fetch uniquement quand l'onglet est activé)
- Filtres chargés une seule fois via `/api/filters`
