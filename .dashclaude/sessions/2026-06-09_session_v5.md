## Decisions techniques
1. **Full sync** (TRUNCATE+INSERT) plutot qu'incremental ‚Äî simple, fiable, idempotent
2. **Failover LAN‚ÜíTailscale** dans db.ts ‚Äî essai LAN (5s timeout), puis Tailscale
3. **COL_ID non AUTO_INCREMENT** ‚Äî apres INSERT, query par COL_NOM pour retrouver la ligne
4. **node-firebird conserve dans sync/** ‚Äî necessaire pour lire le FDB temporaire
5. **electron-store** pour config persistante ‚Äî lu au demarrage, applique au module config
6. **output: standalone** dans next.config ‚Äî necessaire pour le packaging Electron
