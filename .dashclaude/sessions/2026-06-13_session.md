## Décisions prises

1. **Page marque > Page pilotage** : L'historique multi-saisons n'a pas sa place dans le pilotage (qui est scopé à une saison). La page marque est le bon endroit.

2. **RECBRL/RECBR > ARTCOLART pour les ventes** : ARTCOLART associe trop d'articles à chaque collection (appartenance logique vs réception physique). Résultat : taux de sortie aberrants. RECBRL lie les articles aux collections via leurs bons de réception réels.

3. **Scoping des ventes par période** : Même via RECBRL, les ventes all-time gonflent le taux. Solution : CASE WHEN avec dates calculées depuis le nom de collection (PE=jan-jul, AH=jul-jan+1).

4. **Statut basé sur la fin de période** : La dernière vente all-time rendait des saisons anciennes "En cours" si l'article se vendait encore. Fix : `SEASON_TO` calculé depuis COL_NOM, comparé à today.

5. **Taux de sortie = (reçu - stock) / reçu** : Plus fiable que vendu/reçu car les ventes scopées ne captent qu'une partie des écoulements. Le stock restant dit la vérité.

6. **Deux colonnes ventes** : "Vendu saison" (scopé) + "Vendu hors" (total - saison) pour voir la répartition complète.
